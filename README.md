# BVP Solver — Метод продолжения по параметру

Учебная работа по курсу «Научное программирование на Python» (МГУ, 6 семестр).

Приложение для численного решения краевых задач систем обыкновенных дифференциальных уравнений методом продолжения по параметру (метод стрельбы с гомотопией). Реализация по статье: Аввакумов С. Н., Киселев Ю. Н. «Некоторые алгоритмы оптимального управления» (Труды Института математики и механики УрО РАН, 2006).

Графический интерфейс на PyQt6 — две темы (светлая/тёмная), два языка (русский/английский), встроенная библиотека примеров, сохранение и загрузка задач в JSON.

## Математическая постановка

Дана система ОДУ

$$\dot{x} = f(t, x), \quad t \in [a, b],$$

с краевыми условиями

$$R(x(a), x(b)) = 0,$$

где $x \in \mathbb{R}^n$, $R: \mathbb{R}^{2n} \to \mathbb{R}^n$. Число уравнений совпадает с числом краевых условий.

### Метод решения

Задача решается в два уровня.

**Внутренняя задача.** Для текущего кандидата $p = x(t^{*})$ вычисляется невязка

$$\Phi(p) = R\left(x(a; p),\; x(b; p)\right)$$

путём интегрирования из точки $t^{*}$ до концов отрезка расширенной системы — основной траектории $x(t)$ и матрицы вариаций $X(t) = \partial x / \partial p$:

$$\dot{x} = f(t, x), \qquad \dot{X} = f_x(t, x)\, X, \qquad x(t^{*}) = p,\; X(t^{*}) = I.$$

Якобиан правой части $f_x$ вычисляется символьно через SymPy (`Matrix.jacobian`) и компилируется в numpy-функцию (`lambdify`). Это позволяет избежать численного дифференцирования.

**Внешняя задача.** Метод продолжения по параметру (гомотопия). Ищется $p$ такое, что $\Phi(p) = 0$. На каждой внешней итерации $k$ строится решение задачи Коши

$$\frac{dp}{d\mu} = -\left[\Phi'(p)\right]^{-1} \Phi_0, \quad p(0) = p_k, \quad \mu \in [0, 1],$$

где $\Phi_0 = \Phi(p_k)$ — фиксированная невязка на текущем шаге, а $\Phi'(p) = R_{xa} X(a) + R_{xb} X(b)$. Очередное приближение $p_{k+1} = p(1)$. Итерации повторяются пока $\|\Phi(p)\| \geq \varepsilon$ и не превышен лимит итераций.

После сходимости расширенная система интегрируется ещё раз с `dense_output=True` — это даёт интерполятор, по которому строится таблица значений с произвольным шагом.

## Возможности интерфейса

Интерфейс разделён на четыре вкладки.

### Вкладка «Решение»

- **Размерность системы (n)** — спинбокс, поля уравнений / краевых условий / $p_0$ создаются и удаляются динамически.
- **Уравнения** $\dot{x} = f(t, x)$ — переменные `x1..xn`, время `t`.
- **Краевые условия** — обозначения `xa0..xa(n-1)` для $x(a)$ и `xb0..xb(n-1)` для $x(b)$.
- **Отрезок $[a, b]$** и **точка старта** $t^{*}$ (по умолчанию совпадает с $a$).
- **Начальное приближение** $p_0$ — догадка для $x(t^{*})$.
- **Кнопки тулбара:**
  - 📚 Примеры — встроенная библиотека (см. ниже)
  - 📂 Открыть — загрузка задачи из JSON
  - 💾 Сохранить — сохранение задачи в JSON
- **Кнопка «Решить»** — запуск в отдельном потоке (UI не блокируется), статус под кнопкой.

**Карточка графика** содержит:
- 📐 — режим фазовой плоскости (для $n \geq 2$): два комбобокса выбора осей X и Y из переменных $x_1, \ldots, x_n$.
- • — показать/скрыть узлы сетки интегратора (точки на кривой).
- 🎨 — индивидуальная настройка цвета каждой переменной через `QColorDialog`.
- 📏 — ручная установка границ осей (xmin, xmax, ymin, ymax). Пустые поля — autoscale.

**Карточка таблицы** содержит:
- Поле «Шаг $\Delta t$» — равномерный шаг по $t$. Например, при $[a, b] = [0, 1]$ и шаге 0.1 получится 11 строк. Если поле пустое или шаг превышает длину отрезка — таблица строится автопрорежением узлов сетки до ~50 строк.
- Автоматическая нумерация строк.

### Вкладка «Параметры»

Две карточки — **Внутренняя задача** и **Внешняя задача**. В каждой:
- Метод интегрирования — RK45, Radau, LSODA (с человеческими названиями и тултипами).
- Точность — `rtol = atol`, выбор из готового списка ($10^{-2} \ldots 10^{-12}$).
- Внешняя задача дополнительно содержит максимум итераций.

При смене метода точность подставляется автоматически с разумным дефолтом.

### Вкладка «Автор»

Фото, ФИО, группа.

### Вкладка «Помощь»

Семь карточек со скроллом: о программе, ввод данных, параметры, выбор метода, график и таблица, пример, создание собственной задачи (формат JSON), горячие клавиши.

### Темы и языки

- 🌙 / ☀ в правом верхнем углу — переключение светлой и тёмной темы.
- EN / RU — переключение языка интерфейса. Размеры элементов фиксированы, поэтому при смене языка ничего не «прыгает».

## Формат ввода

### Уравнения $\dot{x} = f(t, x)$

Правые части системы в виде символьных выражений. Допустимые имена: `x1`, `x2`, …, `xn`, `t`. Допустимы стандартные функции SymPy (`sin`, `cos`, `exp`, `sqrt`, …).

```
x2
-x1
```

### Краевые условия $R(x(a), x(b)) = 0$

Выражения, обращающиеся в ноль. Обозначения значений на концах отрезка (нумерация с нуля):
- `xa0`, `xa1`, … — это $x_1(a)$, $x_2(a)$, …
- `xb0`, `xb1`, … — это $x_1(b)$, $x_2(b)$, …

```
xa0
xb0 - 1
```

### Начальное приближение $p_0$

Догадка для $x(t^{*})$ — по одному числу в каждое поле.

> Парсинг выражений выполняется через `sympy.parsing.sympy_parser.parse_expr`. Функция `eval` не используется.

## Быстрый старт

Простейшая задача: $x'' = 0$, $x(0) = 0$, $x(1) = 1$. Точное решение — $x(t) = t$.

Сводится к системе порядка 2: $x_1 = x$, $x_2 = \dot{x}$.

| Поле   | Значение     |
|--------|--------------|
| n      | 2            |
| x1' =  | `x2`         |
| x2' =  | `0`          |
| R1 =   | `xa0`        |
| R2 =   | `xb0 - 1`    |
| a      | 0            |
| b      | 1            |
| t*     | 0            |
| p1₀    | 0            |
| p2₀    | 1            |

Результат: прямая $x_1 = t$, константа $x_2 = 1$.

## Библиотека примеров

Доступна по кнопке 📚 на вкладке «Решение». Девять задач, разделённых на две группы.

**Учебные:**

| Идентификатор   | Задача                                            |
|-----------------|---------------------------------------------------|
| `linear`        | $x'' = 0$, $x(0) = 0$, $x(1) = 1$ — решение $x = t$ |
| `exponential`   | $x' = x$, $x(0) = 1$ — решение $x = e^t$          |
| `harmonic`      | $x'' + x = 0$, $x(0) = 0$, $x(\pi) = 0$ — решение $x = \sin t$ |

**Из статьи Аввакумова — Киселева (2006):**

| Идентификатор       | Задача                                                  |
|---------------------|---------------------------------------------------------|
| `two_body_1`        | Задача двух тел, пример 1, $p_0 = [2, 0, -0.5, 0.5]$    |
| `two_body_2`        | То же, второе решение, $p_0 = [2, 0, 0.5, -0.5]$        |
| `ekweiler_1..3`     | Предельные циклы Эквейлера, пример 2 (три случая)        |
| `triple_integrator` | Трёхкратный интегратор со сглаженной функцией `sat`, пример 3 |

## Формат JSON

Любую задачу можно сохранить и загрузить через кнопки 💾 и 📂. Также JSON можно создать вручную в любом текстовом редакторе.

```json
{
  "n": 2,
  "equations": ["x2", "0"],
  "boundary": ["xa0", "xb0 - 1"],
  "a": 0.0,
  "b": 1.0,
  "t_star": 0.0,
  "p0": [0.0, 1.0],
  "inner_method": "RK45",
  "outer_method": "RK45",
  "inner_tol": "1e-6",
  "outer_tol": "1e-4",
  "max_iter": 10
}
```

| Поле                          | Описание                                                     |
|-------------------------------|--------------------------------------------------------------|
| `n`                           | Размерность системы                                          |
| `equations`                   | Список из `n` правых частей $\dot{x} = f(t, x)$              |
| `boundary`                    | Список из `n` краевых условий $R = 0$                        |
| `a`, `b`                      | Границы отрезка интегрирования                               |
| `t_star`                      | Точка старта (обычно равна `a`)                              |
| `p0`                          | Начальное приближение $x(t^{*})$ длины `n`                     |
| `inner_method`, `outer_method`| Один из: `RK45`, `Radau`, `LSODA`                            |
| `inner_tol`, `outer_tol`      | Точность строкой, например `"1e-6"`                          |
| `max_iter`                    | Максимум внешних итераций                                    |

## Методы интегрирования

В solve_bvp используется `scipy.integrate.solve_ivp` с одним из трёх методов.

| Метод     | Тип        | Когда использовать                                                     |
|-----------|------------|------------------------------------------------------------------------|
| **RK45**  | Явный      | Универсальный, дефолт. Нежёсткие задачи: механика, орбиты, колебания. |
| **Radau** | Неявный    | Жёсткие системы: химкинетика, цепи, переключения.                      |
| **LSODA** | Адаптивный | Автоматический выбор жёсткий/нежёсткий. Если природа задачи неясна.    |

Шкала точности (используется как `rtol = atol`):

| Уровень           | Назначение                              |
|-------------------|-----------------------------------------|
| `1e-2` … `1e-3`   | Грубая прикидка, форма решения          |
| `1e-4` … `1e-6`   | Стандарт, дефолт программы              |
| `1e-8` … `1e-9`   | Высокая точность                        |
| `1e-10` … `1e-12` | Научные расчёты                         |

> Эмпирическое правило: точность внутренней задачи должна быть жёстче точности внешней, иначе внешний метод «не видит» реальной ошибки.
